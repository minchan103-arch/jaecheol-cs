# 의사결정 기록

주요 기술적/비즈니스적 선택과 그 이유를 기록합니다.

---

## 2026-03-03

### Claude 모델: claude-sonnet-4-6
- **결정**: `claude-sonnet-4-6` 사용
- **이유**: CS 응답 품질과 응답 속도의 균형, 한국어 자연스러운 처리

### 에스컬레이션 방식: ESCALATE: 접두사
- **결정**: Claude가 `ESCALATE:` 접두사로 시작하는 응답을 보내면 프론트에서 감지
- **이유**: 별도 함수 호출(tool_use) 없이 단순 텍스트 파싱으로 처리 가능. 구현 단순화.

### 카카오톡 에스컬레이션: 로그만 남김
- **결정**: 카카오 채널에서는 에스컬레이션 시 상담사 연결 버튼 없이 로그만 출력
- **이유**: 카카오 i 오픈빌더 구조상 버튼 추가가 복잡함. 운영자가 로그 모니터링.
- **개선 가능**: quickReplies로 상담사 연결 버튼 추가 가능

### Tailwind CSS v4
- **결정**: Tailwind v4 사용 (`@import "tailwindcss"` 방식)
- **이유**: Next.js 최신 템플릿 기본값. `@theme inline`으로 CSS 변수와 Tailwind 테마 통합.

### Google Sheets 로깅: fire-and-forget
- **결정**: 채팅 로그를 Google Sheets에 비동기로 전송 (응답 대기 없음)
- **이유**: 로깅 실패가 사용자 응답에 영향을 주지 않아야 함

### 손상 대응 반품 없음
- **결정**: 신선식품 특성상 반품 요청 안 함 (받아도 폐기)
- **이유**: 물류 비용 절감 + 고객 편의 (반품 절차 없이 재발송/환불)
